# 家計簿 CSV取込（突合）GAS

Googleフォームの回答スプレッドシートに常駐させ、カード／銀行のCSVから
**手入力で拾えなかった支出だけ**を追記する仕組み。二重計上を構造的に防ぐ。

## できること

- イオン明細 / 銀行履歴 / PayPay明細のCSVを**種類自動判定**で取込
- **取込済み台帳（指紋）** で、同じCSVを何度入れても二重にならない
- 既存（手入力含む）と**日付＋金額で突合**し、疑わしい行は**「保留」**へ（自動追記しない）
- 銀行の**カード引落集計行・ATM・入金**は除外し**「除外ログ」**に記録（取りこぼし監査用）
- 口座振替などの**固定費**は「定期費マスター」から月1で自動追記

## セットアップ手順

1. 対象スプレッドシートを開く → メニュー **拡張機能 → Apps Script**
2. 既定の `コード.gs` の中身を [`Code.gs`](./Code.gs) の内容に置き換え
3. 左の「ファイル＋」→ **HTML** を追加し、名前を **`Upload`** にして [`Upload.html`](./Upload.html) の内容を貼り付け
4. 保存 → スプレッドシートを再読み込み
5. メニュー **📥 家計簿取込 → 初期セットアップ（シート作成）** を実行
   - 初回のみ認可ダイアログが出る → 自分のアカウントで承認（外部連携なし・このシートの編集権限のみ）
6. 作成された **取込ルール / 除外ルール / 定期費マスター** を必要に応じて編集

## 使い方

### CSV取込
1. カード会社サイトから明細CSVをダウンロード
2. メニュー **📥 家計簿取込 → CSVを取り込む…**
3. CSVを選ぶ（複数可）→「取り込む」
4. 結果が表示される（自動追記○件／保留○件／除外○件／スキップ○件）

### 保留の確認
- 「保留」シートを開き、**本当に新規の支出だけ**「承認」にチェック
- 必要ならカテゴリを修正
- メニュー **保留の承認分を反映** → 回答シートへ追記、ダッシュボードに反映

### 定期費（口座振替など）
- 「定期費マスター」に 有効/項目/カテゴリ/金額/計上日 を登録（有効=TRUEのみ対象）
- メニュー **定期費を今月分追加**（毎月実行。重複追記はされない）
- ※時間トリガーで毎月自動実行も可能（トリガー → addMonthlyFixed を月次設定）

## 重要な設計（二重計上を防ぐ仕組み）

| ソース | 取り込む | 除外する |
|---|---|---|
| イオン明細 | 各利用明細 | フッタ合計 |
| PayPay明細 | 各利用明細 | 合計・繰越列 |
| 銀行履歴 | Visaデビット・口座振替など（除外リスト以外の引出） | カード引落集計（PayPay/イオン）・ATM・振込・入金 |

- 銀行の「PayPayカード ¥9,173」のような**集計行は取り込まず**、PayPay明細側の個別行を採用 → 二重計上を防止
- 除外した行は **除外ログ** に残るので、「明細CSVを取り込み忘れて丸ごと消える」事故を後から確認できる

## カテゴリについての注意

ルールに当たらない明細は既定で **`未分類`** で追記されます。
ダッシュボードの**カテゴリ別集計は 01〜14 のマスターカテゴリのみ**を対象にするため、
`未分類` の行は月間合計には含まれますが、カテゴリ別グラフには出ません。
→ 「未分類」を定期的に見て 01〜14 に振り直すか、`Code.gs` の `FALLBACK_CATEGORY` を
`'14 臨時出費'` に変更すると常にグラフへ反映されます。

## 調整ポイント（Code.gs 冒頭の CONFIG）

- `RESPONSE_SHEET` … 回答シート名（既定 `家計簿【支出】（回答）`）
- `COL` … 回答シートの列マッピング（フォームの列順が違う場合はここを調整）
- `FALLBACK_CATEGORY` … 未マッチ時のカテゴリ
